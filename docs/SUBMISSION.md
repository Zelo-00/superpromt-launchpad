## Публикация на маркетплейсе — план (оператор, из IDE)

Вкладка **Marketplace** → «Опубликовать» → тип → файлы → title/desc/tags/license/version → ревью Kodik.

1. **Project** — «SuperPromt: PSQ-оценка постановки задач»
   - Desc: «Оценивает качество постановки задачи для LLM по детерминированной метрике PSQ (чеклист CSP-8 + анти-накрутка), доводит промт в пошаговой Студии и подбирает web/backend-скиллы. FastAPI + ванильный SPA, работает без ключа (Fast Mode), BYOK для глубокой оценки.»
   - Tags: `prompt-engineering, psq, fastapi, web, quality-gate`; License: MIT; Version: 1.0.0
   - Приложить: репо/архив + demo-ссылка + скриншоты из docs/screenshots/
2. **Skill** (пак) — «Web/Backend Skill Pack» — 10 SKILL.md из `skills/` (формат Kodik-совместимый)
3. **Command** — «PSQ Prompt Gate» — промт-шаблон оценки постановки по CSP-8 до передачи агенту
4. **Rule** — «Clean Task Statement» — правило: постановки в агент только с закрытым CSP-8
5. **Бонус: MCP** — конфиг `mcp/kodik-mcp-config.json` → PSQ-гейт как `mcp__superpromt__*` (описать в Project)

## Самооценка по 7 критериям (цель — максимум)

1. **Код и архитектура (20).** Слоистость: stdlib-ядро → FastAPI-обёртка → SPA; логика не дублируется; conventional commits (30+), CI (pytest), нет «всего в одном файле». ✅
2. **Продукт и инженерная элегантность (20).** End-to-end флоу работает без ключа (детерминированный prescreen — «просто там, где нужно просто»); BYOK-глубина — там, где оправдано; MCP-интеграция — технически смелый ход. ✅ (после QA-отчёта закрыть найденные дефекты)
3. **Идея и коммерческая ценность (20).** Боль: кредиты/токены сгорают на плохих постановках. ЦА: пользователи агентных IDE, промт-инженеры, новички. Рост на маркетплейсе: Project + Skill + Command + Rule + MCP. README «Для кого это». ✅
4. **Презентация (10).** Статья (ключевой сценарий, роль Kodik, экономика кредитов) + видео 3 мин по сценарию. 🔶 записать видео, опубликовать
5. **UX/дизайн (10).** Авторская айдентика «Молтен-Слэб» (SVG, без эмоджи), онбординг-тур, Студия-коуч, 2 темы, RU/EN, самохостинг шрифтов. ✅
6. **Безопасность (10).** Rate-limit, security-заголовки, лимит длины (422), CORS через env, параметризованный SQL, BYOK только env, .env.example, graceful 503, секрет-скан истории git — чисто. ✅
7. **Документация (10).** README: архитектура, quickstart (venv+Docker), API-таблица, стек, скриншоты, demo-ссылка; MCP README; отчёт о разработке (.docx). ✅
