# SuperPromt: как мы построили PSQ-гейт для промтов в AI-IDE Kodik — и почему плохая постановка задачи стоит дороже, чем кажется

*Статья для Habr / vc.ru (Kodik Launchpad, июль 2026). Черновик — оператор дополняет личными деталями и публикует со своего аккаунта.*

---

Каждый, кто работает с ИИ-агентами — в Kodik, Claude Code, Cursor — знает этот цикл: написал задачу, агент понял по-своему, три итерации правок, кредиты сгорели, результат «почти то». Проблема почти никогда не в модели. Проблема в постановке.

Мы сделали **SuperPromt** — веб-приложение, которое оценивает качество постановки задачи *до* того, как вы потратите на неё токены, и доводит её до состояния, из которого агент собирает результат с первого прогона.

## Проблема: «сделай сайт» — это не задача

LLM-агенты детерминированно хороши ровно настолько, насколько хороша инструкция. Но «качество инструкции» обычно оценивают на глаз. Мы формализовали его в метрику:

**PSQ (Prompt Specification Quality)** = (Σ cᵢ / 8) · (1 − 0.5·a)

- **CSP-8** — чеклист из 8 пунктов, которые должна закрывать постановка: предметная область, целевая операция, формат выхода, критерии качества, контекстные якоря, роль, пример, язык ответа;
- **a** — двусмысленность формулировок;
- порог «идеальной» постановки — **0.80**.

Ключевое: метрика **детерминированная и анти-накруточная**. Плейсхолдеры, самохвала («я лучший в мире эксперт»), тавтологии («формат: соблюдён») не просто не засчитываются — watchdog режет за них балл. Заведомо слабые и накрученные постановки отбраковываются prescreen-ом **без единого обращения к LLM** — то есть бесплатно.

## Что умеет продукт

- **Оценка PSQ** — вставили задачу, получили балл на анимированном gauge, разбор по CSP-8 (что закрыто, что нет), флаги накрутки и рекомендованные скиллы под задачу.
- **Prompt Studio** — пошаговый коуч: 8 экранов, живой предпросмотр, на выходе — промт, закрывающий весь чеклист. Это ещё и обучалка: пару прогонов — и вы пишете такие постановки сами.
- **Сравнение A/B** — два варианта постановки бок о бок с вердиктом.
- **Аналитика** — тренд PSQ, распределение флагов, доля слабых постановок; виден собственный прогресс.
- **История** — SQLite-хранилище всех оценок.
- **BYOK** — свой ключ OpenAI/Anthropic/Google включает «глубокую» оценку LLM-судьёй; без ключа честный Fast Mode (и честный 503 вместо выдуманного балла).
- **RU/EN, тёмная/светлая тема, экспорт JSON/Markdown, шеринг-карточки** на Canvas.

Под капотом — три слоя, и это осознанная архитектура:

```
superprompt_cli/  ядро: PSQ, watchdog, router, TF-IDF-подбор скиллов (только stdlib)
web/              FastAPI-обёртка над ядром + ванильный SPA
mcp/              MCP-сервер: PSQ-гейт внутри самого Kodik (stdio JSON-RPC 2.0)
```

Веб-слой не дублирует ни строчки доменной логики — только оборачивает ядро в HTTP. Ядро и MCP-сервер — ноль внешних зависимостей.

## Как мы использовали Kodik

Весь веб-слой построен в AI-IDE Kodik, и это отдельная история про экономику агентной разработки.

- **Агент с планированием и чекпоинтами** собирал фичи целиком: бэкенд-эндпоинты, историю на SQLite, аналитический дашборд, Prompt Studio, режим сравнения, i18n, экспорт. Одна содержательная фича — одна задача агенту — несколько кредитов.
- **Автоподтверждение правок** (с блок-листом на опасные команды — `rm` туда попал не зря) превращает агента в конвейер: формулируешь → уходишь смотреть диф следующей фичи.
- **Интегрированный терминал и встроенный браузер** — pytest и curl-проверки агент гонял сам, там же, где правил код.
- **KodikShield** маскировал секреты (ключи провайдеров, учётные данные) до отправки контекста в облако — для проекта, который сам работает с BYOK-ключами, это гигиенический минимум.
- **QA отдельными задачами**: финальный прогон — агент-тестировщик с жёсткой инструкцией «ничего не править, только отчёт о дефектах с приоритетами».

Из ~250 кредитов тарифа на весь веб-слой ушло меньше половины — потому что каждая задача уходила агенту уже с высоким PSQ. Да, мы ели свой же корм: постановки для агента прогонялись через SuperPromt.

## Ключевой пользовательский сценарий

1. Открываете SuperPromt, вставляете черновик задачи: «сделай REST API для заметок».
2. PSQ 0.30, красная зона: не закрыты формат выхода, критерии качества, роль, пример. Флагов накрутки нет.
3. Жмёте «Хотите сэкономить токены?» → Prompt Studio ведёт по 8 шагам, предпросмотр собирает промт на глазах.
4. Итог: PSQ 0.85+, зелёная зона + три релевантных скилла (fastapi-backend, rest-api-design, sql-postgres) — копируете кнопкой.
5. Отдаёте постановку агенту в Kodik — результат с первого прогона. В «Аналитике» виден ваш тренд: слабых постановок с каждой неделей меньше.

## Где это жить будет

Продукт публикуется на маркетплейсе Kodik в трёх формах: **Project** (веб-приложение), **Skill-пак** (10 курированных web/backend-скиллов) и **MCP-сервер** — PSQ-гейт и подбор скиллов доступны любому агенту Kodik как `mcp__superpromt__*` инструменты.
