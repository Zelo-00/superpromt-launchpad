## Оценка по 7 критериям (макс. 100)

### 1. Качество кода и архитектура (0–20)
Слоистость: **stdlib-ядро `superprompt_cli`** (PSQ, router, webskills, providers) → **FastAPI-обёртка**
(`web/backend/main.py`, типизированные Pydantic-модели, эндпоинты по одному ответственному действию) →
**ванильный SPA** (`web/frontend`). Логика ранжирования/оценки не дублируется между CLI и вебом.
Чистые функции выделены (`_answer_to_files`, `_verify_result`). Нет «всё в одном файле».
Самооценка: **17–18/20**.

### 2. Продуктовая реализация и инженерная элегантность (0–20)
End-to-end флоу работает публично: промт → PSQ → скиллы → **сборка проекта агентами** → живое
превью в iframe + ZIP. Решение точное под задачу: детерминированная PSQ там, где не нужен LLM;
агентная проверка + **ремонт-цикл до результата** там, где важно качество. Дёшево и надёжно —
`gpt-oss-20b` (самая дешёвая) стабильно даёт PASS благодаря конвейеру. Самооценка: **18/20**.

### 3. Идея и коммерческая ценность (0–20)
Решает реальную боль: расплывчатые промты → плохой результат и слив токенов. PSQ измеряет
постановку ДО траты, конвейер собирает готовый артефакт. Аудитория — разработчики, ноукод,
агентства. Потенциал на маркетплейсе Kodik: Project + Skill-пак + Command + Rule. **16–17/20**.

### 4. Презентация проекта (0–10)
Статья + сценарий видео готовы (`docs/`), нужен публичный пост. Ключевой сценарий — ниже. **7–8/10**
(после публикации видео/статьи в открытом доступе — до 10).

### 5. UX / дизайн (0–10)
Понятный однопанельный поток, живой PSQ-гейдж, наглядный **процесс агентов** (стадии Router→…→ревьюер),
превью результата, тёмная/светлая темы, RU по умолчанию + EN. **8–9/10**.

### 6. Безопасность и аккуратность (0–10)
Секреты не в конфиге — только ссылки (`~/.spt/.env`, права 0600); превью-эндпоинт защищён от
path-traversal (confine внутри каталога генерации); `proxy_headers=False` (анти-подмена X-Forwarded-For);
валидация длины промта; sandbox на iframe превью. **8–9/10**.

### 7. Документация (0–10)
README (стек, возможности, таблица моделей), инструкции запуска (Docker), скриншоты,
эта самооценка, чеклист сдачи `docs/SUBMISSION.md`. **8–9/10**.

**Итоговая самооценка: ~82–89 / 100.** Оставшиеся баллы закрываются публикацией на маркетплейсе
и публичного видео/статьи (условия допуска и критерий 4).

## Ключевой пользовательский сценарий (для видео/защиты)
1. Открыть https://superpromt.site — ввести расплывчатый промт «сделай сайт кофейни».
2. PSQ показывает низкий балл (постановка слабая) и рекомендует скиллы.
3. Нажать **«🛠 Собрать архив»** → на экране идёт **процесс агентов**: Router → PSQ-гейт
   (0.21 → 1.0) → подбор скиллов → генератор → **агент-ревьюер (PASS 6/6)**.
4. Появляется **живое превью** готового сайта + кнопка скачать ZIP (index.html + css + js + launch.bat).
5. Показать независимость: тот же промт на `qwen3-next-80b` / `qwen3-coder-480b` — тоже PASS.

## Что уникально
- **Две метрики раздельно** остаются в философии, но в этом веб-продукте KSCR заменён на
  **проверку агентами** (детерминированные чеки + LLM-ревьюер + ремонт-цикл) — практичнее для
  «собери и покажи».
- **Не зависит от одной модели** — конвейер вытягивает результат даже на самой дешёвой `gpt-oss-20b`.
