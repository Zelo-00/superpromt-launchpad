## Когда использовать
Настраиваешь конфигурацию приложения и работу с ключами/секретами.
## Ключевые практики
- 12-factor: вся конфигурация — из окружения. Код одинаков для dev/stage/prod, различия — в env.
- `.env` — локально и в `.gitignore`; в репозитории — `.env.example` без значений. Секреты прода — в секрет-сторе (Vault/облачный).
- Типизированная загрузка конфига (pydantic-settings/zod-env) с валидацией на старте — падать рано при отсутствии обязательных.
- Разные ключи/БД на среду; ротация ключей; не логируй секреты.
## Анти-паттерны
- Ключи в коде/репозитории/фронтенд-бандле, один секрет на все среды, конфиг «магическими» глобалами.
