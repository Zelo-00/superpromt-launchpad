## Когда использовать
Проектируешь схему БД или пишешь/оптимизируешь SQL (Postgres).
## Ключевые практики
- Нормализуй до 3НФ, денормализуй осознанно под чтение. PK — `bigint generated always as identity` или uuid.
- Индексы под реальные запросы (WHERE/JOIN/ORDER BY); проверяй `EXPLAIN ANALYZE`. Составные индексы — по порядку селективности.
- Внешние ключи + `ON DELETE` явно. `NOT NULL` и `CHECK` — на границе данных.
- Транзакции для многошаговых изменений; уровень изоляции по нужде.
- Миграции — версионируемые (alembic/knex/prisma), никогда не правь схему руками на проде.
- Против N+1 — JOIN/`IN`/dataloader, не цикл запросов.
## Анти-паттерны
- `SELECT *` в проде, отсутствие индексов на FK, строковая конкатенация SQL (→ инъекции), схема без миграций.
