# Chart.js / ECharts — дашборды и визуализация данных

## Когда использовать
Дашборд метрик (KPI-строка + графики), временной ряд, сравнение категорий, распределение,
live-обновление данных, отчёт с графиками. НЕ использовать: диаграммы процессов/архитектуры
(→ mermaid-diagrams), декоративные фоны (→ paper-shaders), научные фигуры для статей (→ math-orchestr).

## Ключевые практики

### Выбор типа графика под данные
- Динамика во времени → line (area — только если важна накопленная величина).
- Сравнение категорий → bar горизонтальный при длинных подписях, вертикальный при ≤7 категориях.
- Доля целого → stacked bar / donut (≤5 сегментов, иначе «прочее»); pie избегать при близких долях.
- Связь двух величин → scatter (+линия тренда); распределение → histogram; матрица → heatmap.
- Один показатель «сейчас» → KPI-плитка числом (не спидометр).

### Каркас дашборда
- Сетка: KPI-строка сверху (4–6 плиток: значение + дельта к прошлому периоду + спарклайн),
  ниже 2-колоночная сетка графиков; каждый график в карточке с заголовком-вопросом
  («Откуда приходят пользователи?», не «Источники»).
- Адаптив: `grid-template-columns: repeat(auto-fit, minmax(320px,1fr))`; canvas в контейнере
  с `position:relative; height:` — Chart.js `maintainAspectRatio:false`.

### Chart.js базис
- `new Chart(ctx, {type, data:{labels, datasets}, options})`; общие options выносить в
  `Chart.defaults` (шрифт, цвета сетки) — единый вид всех графиков.
- Оси: y от нуля для bar (иначе враньё масштабом); `ticks.callback` для форматов (₽, %, k/M);
  время — адаптер date-fns, `type:'time'`.
- Тултипы: общий `interaction:{mode:'index', intersect:false}` на line — сравнение серий в точке.
- Легенда: снизу/справа, кликабельна (скрытие серии) — оставлять включённой при ≥2 сериях.

### Цвет и темы
- 6–8 категориальных цветов с контрастом к фону ≥3:1, различимых при дальтонизме
  (сине-оранжевая база, не красный-зелёный); последовательные шкалы — один тон от светлого к тёмному.
- Тёмная тема: цвета сетки/текста из CSS-переменных, у графиков полупрозрачные заливки
  (`rgba`), обновление `chart.options` при переключении темы + `chart.update()`.

### Живые данные
- Поллинг/SSE → `chart.data.datasets[0].data.push(p); chart.data.labels.push(t);` кап окна
  (сдвиг `shift()` при > N точек) → `chart.update('none')` (без анимации — не дёргается).
- Скелетон/спиннер до первых данных; пустое состояние с текстом, не пустые оси.

## Анти-паттерны
- 3D/тени/градиенты «для красоты» — искажают чтение; anima­ция > 300 мс на каждом апдейте.
- Два разных y-масштаба без явной маркировки; y не от нуля у столбцов.
- >7 линий на одном графике (дай выбор серий); подписи осей под 45° (лучше горизонтальный bar).
- Цвет как единственный канал различия серий (добавь маркеры/штрих для печати и дальтоников).
