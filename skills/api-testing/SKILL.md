## Когда использовать
Добавляешь тесты к сервису/эндпоинтам (обязательно перед «готово»).
## Ключевые практики
- Пирамида: много unit (логика/services), меньше integration (эндпоинт+БД), немного e2e.
- Python — pytest + httpx/TestClient; Node — jest/vitest + supertest.
- Тест-БД: отдельная схема/контейнер, откат транзакции или пересоздание между тестами; фикстуры вместо ручной подготовки.
- Проверяй happy-path И границы: валидация (400/422), права (401/403), 404, конфликты.
- Мокай внешние сети/оплату; критичный путь оставляй интеграционным.
## Анти-паттерны
- Тесты, зависящие от порядка/общего состояния, только happy-path, реальные внешние вызовы в CI.
