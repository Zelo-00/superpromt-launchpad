## Когда использовать
Проектируешь новый HTTP API или ревьюишь существующий эндпоинт.
## Ключевые практики
- Ресурсы — существительные во множественном числе: `/users`, `/orders/{id}/items`. Действия — через методы, не в URL.
- Методы: GET (чтение, идемпотентно), POST (создание), PUT (полная замена, идемпотентно), PATCH (частичное), DELETE.
- Статусы: 200/201/204 успех, 400 валидация, 401 не аутентифицирован, 403 нет прав, 404 нет ресурса, 409 конфликт, 422 семантика, 429 лимит, 5xx сервер.
- Пагинация: cursor-based для больших наборов (`?cursor=&limit=`), не offset на горячих таблицах.
- Версионируй (`/v1/`) с первого дня. Ошибки — единым форматом `{error:{code,message,details}}`.
- Идемпотентность записи — через `Idempotency-Key` заголовок для POST оплат/заказов.
## Анти-паттерны
- Глаголы в URL (`/getUser`, `/createOrder`), 200 на ошибку с `{success:false}`, возврат стектрейсов клиенту.
