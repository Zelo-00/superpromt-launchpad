## Когда использовать
Любой эндпоинт/форма, принимающая пользовательский ввод.
## Ключевые практики
- Валидируй на границе доверия схемой (zod/pydantic/joi): типы, длины, форматы, whitelisting значений.
- SQL — только параметризованные запросы/ORM. Никогда не конкатенируй ввод в SQL/команды.
- XSS — экранируй вывод в HTML, `Content-Security-Policy`, не вставляй сырой ввод в DOM.
- CSRF — SameSite-cookie + токен для форм; CORS — явный whitelist origin.
- Лимиты — размер тела, rate-limit, тайм-ауты; файлы — проверка типа/размера.
- Секреты — в env/секрет-стор, не в коде/логах/репозитории.
## Анти-паттерны
- Доверие клиентской валидации, `eval`/шелл с пользовательским вводом, `cors: *` с cookie, стектрейсы клиенту.
