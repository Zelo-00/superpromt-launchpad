## Когда использовать
Добавляешь вход/регистрацию, защиту эндпоинтов, роли.
## Ключевые практики
- Пароли — только хеш (argon2id/bcrypt, cost по железу), никогда не в открытом виде и не в логах.
- Сессии (httpOnly+Secure+SameSite cookie) для веба — проще и безопаснее; JWT — для stateless/межсервисного.
- JWT: короткий access (мин.) + refresh (ротация, отзыв через whitelist/deny-list), подпись RS256/HS256 с секретом из env. Проверяй `exp`, `aud`, `iss`.
- Авторизация ≠ аутентификация: проверяй права на КАЖДОМ защищённом действии (RBAC/ownership), не только «залогинен».
## Анти-паттерны
- Пароль в plain/MD5, JWT в localStorage (XSS), бессрочные токены, `alg:none`, проверка роли на клиенте.
