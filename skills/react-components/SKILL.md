## Когда использовать
Пишешь или ревьюишь React-компоненты.
## Ключевые практики
- Функциональные компоненты + хуки. Один компонент — одна ответственность; выноси логику в кастомные хуки (`useX`).
- Состояние: локальное `useState`, серверное — react-query/SWR (кэш, повторы), глобальное — context/zustand только по нужде.
- Списки — стабильный `key` (id, не index). Мемоизация (`useMemo`/`memo`) только по замеренной проблеме.
- Побочные эффекты в `useEffect` с корректными зависимостями и cleanup.
- Доступность: семантические теги, `label` к инпутам, фокус-состояния, `aria-*` где нужно.
## Анти-паттерны
- Пропс-дриллинг вместо композиции/контекста, index как key, огромные компоненты, эффекты без cleanup.
