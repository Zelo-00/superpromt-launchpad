# Math.md — научный оркестр агентов (протокол ~/Math.md)

Применяй для задач: математическое доказательство/вывод, ML/AI-исследование, дизайн эксперимента,
статистический анализ (анти-p-hacking), научная статья/рецензия, воспроизводимость результатов.

## Каскад ролей (по протоколу ~/Math.md)
1. **task-architect** — K-оптимизация постановки: формализовать задачу, критерии успеха, ограничения.
2. **math-professor** — строгий вывод: определения → леммы → теорема → доказательство; каждая формула
   перепроверяется подстановкой/численно.
3. **ml-ai-engineer** — реализация/эксперимент: код, датасеты, абляции; фиксация seed и версий.
4. **synthetic-analyst** — синтез результатов разных ролей, поиск противоречий.
5. **kscr-controller** — ИЗОЛИРОВАННЫЙ аудит: K (полнота), S (качество), G=K−S, η=S/K, C=√(K·S);
   галлюцинация = G<0 или η>1.2; PASS при C≥0.80, G≤0.10, 0.90≤η≤1.10. Только аудит, не правки.
6. **reproducibility-verifier** — повторный прогон с нуля: результат обязан воспроизводиться.
7. **experiment-statistician** — методология: мощность, поправки на множественность, эффект-сайзы.
8. **peer-reviewer** — новизна/значимость/слабые места как внешний рецензент.

## Дисциплина
- Вычислимое — ПЕРЕСЧИТАТЬ (не «выглядит верно»); ссылки/DOI — проверить реальным запросом.
- Недоказанное помечать «гипотеза»; отрицательный результат фиксировать честно.
- Цикл: постановка → исполнение → KSCR-гейт → ремонт → … до PASS или честного ABSTAIN.
- Полный протокол ролей и фаз: `~/Math.md`; определения агентов: `~/.claude/agents/*.md`.
