# Godot 4 — разработка игр

## Когда использовать
Создание 2D/3D игры (платформер, аркада, головоломка, top-down), прототип геймплея,
игровые механики (инвентарь, диалоги, сохранения), экспорт под Windows/Linux/Web/Android.
НЕ использовать: браузерная мини-игра на чистом canvas без движка (→ frontend-design + JS),
симуляция/визуализация данных (→ dashboards).

## Ключевые практики

### Структура проекта
- Сцена = единица композиции: `Player.tscn`, `Enemy.tscn`, `Level01.tscn`, `UI.tscn`;
  главная сцена — тонкий корень, собирающий уровень + UI + менеджеры.
- Каталоги: `scenes/` `scripts/` `assets/` (sprites/audio/fonts) `autoload/` (синглтоны).
- Autoload-синглтоны только для сквозного: `GameState` (очки, прогресс), `AudioManager`,
  `SceneLoader`. Не сваливать игровую логику в глобалы.

### Ноды и сигналы
- Композиция вместо наследования: `CharacterBody2D` + `Sprite2D` + `CollisionShape2D` +
  `AnimationPlayer`; поведение — компонентами-нодами.
- Связь ВВЕРХ — сигналами (`signal died(cause)` → уровень слушает), связь ВНИЗ — прямые
  вызовы; «вбок» — через общего родителя или EventBus-синглтон. Нода не знает своих потребителей.
- `@onready var sprite: Sprite2D = $Sprite2D` — кэшировать ссылки, типизировать (GDScript 2.0).

### Движение и физика (2D-платформер)
- `_physics_process(delta)` + `move_and_slide()`; гравитация из ProjectSettings.
- Отзывчивость: coyote time (~0.1 c после схода с платформы), jump buffering (~0.15 c до
  приземления), переменная высота прыжка (отпустил — режь скорость вверх).
- Слои/маски коллизий именовать (player/enemy/world/pickup); Area2D — для триггеров.

### Состояние игры
- Машина состояний игрока: enum + match в `_physics_process` (IDLE/RUN/JUMP/FALL/HURT) или
  ноды-состояния для сложного; анимации через `AnimationTree` + StateMachine.
- Сохранения: словарь → `JSON.stringify` → `user://save.json` (или `ResourceSaver` для типизированных);
  версия формата в файле, миграция при загрузке.

### UI и меню
- CanvasLayer для HUD; Control-ноды с якорями (не абсолютные позиции); темы — `Theme`-ресурс.
- Пауза: `get_tree().paused = true` + `process_mode = PROCESS_MODE_WHEN_PAUSED` у меню паузы.

### Экспорт и оптимизация
- Экспорт-пресеты per-платформа; для Web — сжатие текстур, музыка ogg; для мобильных — ETC2.
- Пул объектов для пуль/частиц (не instantiate/free каждый кадр); `VisibleOnScreenNotifier2D`
  выключает офскрин-логику; статичная геометрия — в один TileMap/MultiMesh (draw calls).

## Анти-паттерны
- Логика в `_process` там, где нужна физика (`_physics_process`) — дрожание на разных FPS.
- `get_node("../../..")` — хрупкие пути вверх; исполь­зуй сигналы/экспортированные NodePath.
- Гигантский `Main.gd` на всю игру; отсутствие пула у часто спавнящихся объектов.
- Магические числа в коде вместо `@export var speed := 300.0` (настройка из инспектора).
