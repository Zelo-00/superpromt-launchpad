# План разработки веб-приложения SuperPromt (исторический документ)

> Первоначальный план Шага 1. Реальность превзошла его: фронтенд осознанно сделан на ванильном JS (см. README «Стек»), все пункты Шагов 1–3 выполнены.

Перенос CLI-конвейера оценки промтов (PSQ) в веб-интерфейс.

## Архитектура
- **Backend**: FastAPI (Python 3.10+).
- **Frontend**: React/Next.js или Vue (в следующих итерациях).
- **Интеграция**: Прямой импорт логики из `superprompt_cli`.

## Шаг 1: Бэкенд (Текущий этап)
- [ ] Создание структуры `web/backend/`.
- [ ] Реализация эндпоинтов:
    - `GET /api/health`: Проверка работоспособности.
    - `POST /api/psq`: Оценка сырой задачи (PSQ score, разбор CSP-8, флаги watchdog).
    - `GET /api/skills`: Получение топ-скиллов для конкретной задачи.
- [ ] Валидация входных данных через Pydantic.
- [ ] Настройка CORS для будущего фронтенда.
- [ ] Конфигурация через `.env` (модели LLM, API ключи).

## Шаг 2: Фронтенд (Будущее)
- [ ] Интерфейс ввода промта.
- [ ] Визуализация PSQ-балла (прогресс-бар, радарная диаграмма CSP-8).
- [ ] Список рекомендованных скиллов с возможностью копирования.
- [ ] Кнопка "Repair" для автоматического улучшения промта.

## Шаг 3: Деплой
- [ ] Dockerfile для бэкенда.
- [ ] Docker Compose для связки фронтенд + бэкенд.
